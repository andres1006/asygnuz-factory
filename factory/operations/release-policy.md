# Release Policy

## Tipos de release
- Semanal estándar
- Hotfix crítico

## Requisitos mínimos release semanal
- Gate 1..7 aprobados
- UAT sin bloqueantes
- Seguridad sin hallazgos críticos abiertos

## Versionado recomendado
- Semantic versioning para producto
- Versionado del template por separado
