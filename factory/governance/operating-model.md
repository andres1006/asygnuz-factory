# Operating Model - POC Factory

## Propósito
Construir productos de software validables en ciclos semanales, con calidad técnica alta, trazabilidad total y aprendizaje acumulativo para mejorar el template.

## Principios operativos
1. Entrega de valor semanal > perfección tardía.
2. DB-first para claridad y escalabilidad.
3. Trazabilidad obligatoria de extremo a extremo.
4. Calidad no negociable (tests + UAT + seguridad).
5. Mejora continua: cada proyecto retroalimenta el template.

## Cadencia híbrida
- Planeación ligera semanal (lunes)
- Ejecución continua (kanban diario)
- Demo + UAT semanal (jueves/viernes)
- Retrospectiva técnica y de proceso (viernes)

## Flujo macro
Discovery -> Producto -> Diseño -> Arquitectura/DB -> Desarrollo -> QA -> UAT -> Release -> Aprendizaje -> Actualización template
